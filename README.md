# ESPHome Electrolux EWH Centurio IQ 3.0

> ⚠️ **Бета-версия.** Проект находится в активной разработке. Возможны баги и ошибки. Используйте на свой страх и риск.

Кастомный компонент ESPHome для управления электрическим водонагревателем через Home Assistant. Разработан для **Electrolux EWH Centurio IQ 3.0** с платой управления **94VO E487430 SM-D** и протестирован на **Electrolux EWH 80 Centurio IQ 3.0**

---

## Возможности

- Полное управление из Home Assistant и физическими кнопками
- 3 ступени мощности нагрева (0.7 / 1.3 / 2.0 кВт)
- Режим антизамерзания (nF) с гистерезисом 4°C/5°C
- Режим дезинфекции воды BST (Bacteria Stop Technology)
- Таймер включения бойлера в заданное время
- Ручная установка текущего времени (для работы без интернета)
- Автозапуск BST по расписанию (день недели + час)
- Аварийная защита с автоматическим отключением
- Подсчёт потреблённой электроэнергии

## Логика управления

### Включение / выключение

Бойлер управляется двумя независимыми каналами:

**Физический** — концевик крутилки (GPIO4). Поворот ручки до упора в положение OFF выключает бойлер аппаратно. При выключении концевиком все режимы (BST, таймер, антизамерзание) немедленно сбрасываются.

**Электронный** — через Home Assistant. Можно включить/выключить даже при выключенном концевике.

При старте ESP32 компонент автоматически читает состояние концевика и восстанавливает режим работы.

---

### Ступени мощности

Три ступени управляются двумя реле:

| Ступень | Реле 1 (0.7 кВт) | Реле 2 (1.3 кВт) | Итого |
|---------|:-:|:-:|-------|
| ECO | ✓ | — | 0.7 кВт |
| COMFORT | — | ✓ | 1.3 кВт |
| BOOST | ✓ | ✓ | 2.0 кВт |

**Физическое управление:** короткое нажатие кнопки Мощность циклично переключает ступени: ECO → COMFORT → BOOST → nF → ECO.

**Из HA:** через пресеты климата (Eco / Comfort / Boost / Away).

---

### Термостатная логика (гистерезис)

Компонент использует классический гистерезисный термостат:

- Нагрев **включается** когда температура падает ниже `уставка - deadband` (по умолчанию 1.0°C)
- Нагрев **выключается** когда температура поднимается выше `уставка + overrun` (по умолчанию 0.5°C)

Уставка задаётся крутилкой (35–75°C) или из HA. Сохраняется в NVS и переживает перезагрузку.

---

### Режим антизамерзания (nF)

Активируется кнопкой Мощность (BOOST → nF) или пресетом "Away" из HA.

В этом режиме:
- Климат переходит в OFF (термостат не управляет реле)
- Реле управляются отдельной логикой с фиксированными порогами:
  - температура < **4.0°C** → оба реле включаются (2.0 кВт)
  - температура ≥ **5.0°C** → реле выключаются
- На дисплее мигает ` nF ` и текущая температура

Выход из режима: кнопка Мощность (→ ECO) или пресет из HA.

---

### BST — Bacteria Stop Technology

Режим дезинфекции воды. Уничтожает легионеллу и другие бактерии путём длительного нагрева.

**Запуск:**
- Физически: зажать кнопку Мощность на 3 секунды
- Из HA: переключатель "BST (Bacteria Stop)"
- Автоматически: по расписанию (настраивается день недели и час)

**Алгоритм цикла:**
1. Сохраняется текущая уставка пользователя
2. Уставка принудительно устанавливается на **70°C**
3. Каждую секунду проверяется температура:
   - если температура ≥ **69.5°C** — счётчик удержания увеличивается
   - если температура падает ниже — счётчик сбрасывается в 0
4. При накоплении **1200 секунд** непрерывного удержания (20 минут) — цикл завершён
5. Уставка восстанавливается до сохранённого значения

**Ограничения:**
- BST нельзя включить при выключенном бойлере
- Взаимоисключение с режимом nF
- Если во время BST крутить крутилку — новая температура сохраняется как уставка "после цикла"

**Дисплей:** ` Sc ` во время активного цикла.

**Аварийное отключение BST:** если цикл длится более 4 часов без достижения температуры — вероятна неисправность ТЭНа. Система аварийно выключается.

---

### Таймер включения

Позволяет включить бойлер в заданное время суток — например чтобы к утреннему душу вода была горячей.

**Установка времени таймера:** зажать кнопку Таймер на 5 секунд → на дисплее мигает HH:MM.
- Кнопка Мощность — изменение часов (+1, зажатие — быстро каждые 500 мс)
- Кнопка Таймер — изменение минут (+1, зажатие — быстро каждые 250 мс)
- Таймаут 10 секунд без нажатий — время сохраняется автоматически
- Поворот крутилки — сохраняет и выходит из режима установки

**Взведение таймера:** короткое нажатие кнопки Таймер при включённом бойлере.

**Поведение при взведённом таймере:**
- Реле не работают (бойлер не греет)
- В HA отображается "Бездействует"
- Дисплей чередует текущее время и время таймера каждые 2 секунды
- В заданное время бойлер начинает нагрев
- После достижения уставки температуры — бойлер **полностью выключается**

**Отмена таймера:**
- Повторное короткое нажатие кнопки Таймер
- Выключение бойлера концевиком или из HA

---

### Установка текущего времени

Используется как резервный способ при отсутствии интернета. При наличии WiFi время синхронизируется автоматически через NTP и записывается в RTC.

**Активация:** зажать одновременно обе кнопки (Мощность + Таймер) на 800 мс → дисплей показывает мигающий HH:MM.

Управление аналогично установке таймера. После применения время записывается в RTC PCF85063 и сохраняется даже при отключении питания.

---

### Аварийная защита

Компонент постоянно мониторит состояние системы. При обнаружении аварии:
1. Реле немедленно отключаются
2. Бойлер переводится в режим OFF
3. BST и таймер сбрасываются
4. В HA публикуется статус с описанием аварии

| Авария | Действие |
|--------|----------|
| Датчик температуры недоступен (NaN) | Аварийное отключение |
| Перегрев > 85°C | Аварийное отключение |
| BST греет > 4 часов без результата | Аварийное отключение |
| Время не синхронизировано | Только предупреждение |

После устранения причины аварии бойлер нужно включить вручную.

---

### Подсчёт энергопотребления

| Сенсор | Описание |
|--------|----------|
| Текущая мощность | 0 / 700 / 1300 / 2000 Вт |
| Энергия за сегодня | Wh, сбрасывается в полночь |
| Энергия всего | kWh, накапливается, кнопка сброса в HA |

Значения сохраняются в NVS и переживают перезагрузку ESP32. Публикуются в HA раз в 10 секунд.

## Сущности в Home Assistant

### Климат
- **Термостат Бойлера** — основная сущность, управление режимом и уставкой

### Переключатели
- **BST (Bacteria Stop)** — вкл/выкл дезинфекции
- **ТЭН 0.7 кВт** — состояние реле 1
- **ТЭН 1.3 кВт** — состояние реле 2

### Сенсоры
- **Water Temperature** — температура воды (°C)
- **Текущая мощность** — мощность нагрева (Вт)
- **Энергия за сегодня** — потребление за день (Вт·ч)
- **Энергия всего** — суммарное потребление (кВт·ч)
- **WiFi сигнал** — уровень сигнала (dBm)
- **Статус системы** — текстовый статус / описание аварии

### Настройки расписания BST
- **BST час запуска** — час автозапуска (0–23)
- **BST день недели** — день автозапуска

### Кнопки
- **Перезагрузка** — перезагрузка ESP32
- **Сбросить счётчик энергии** — обнуление накопленной энергии

## Параметры компонента

```yaml
climate:
  - platform: boiler_climate
    sensor: water_temp           # датчик температуры (обязательно)
    relay_07kw: relay_07kw       # реле 0.7 кВт (обязательно)
    relay_13kw: relay_13kw       # реле 1.3 кВт (обязательно)
    display: boiler_display      # дисплей TM1650 (обязательно)
    click_switch: click_switch   # концевик крутилки (обязательно)
    power_button: power_switch   # кнопка мощности (обязательно)
    timer_button: timer_switch   # кнопка таймера (обязательно)
    potentiometer: potentiometer_raw  # потенциометр (обязательно)
    bst_hour: bst_hour           # number для часа BST (опционально)
    bst_weekday: bst_weekday     # select для дня BST (опционально)
    status_sensor: boiler_status # text_sensor статуса (опционально)
    power_sensor: boiler_power   # сенсор мощности (опционально)
    energy_today_sensor: boiler_energy_today  # энергия сегодня (опционально)
    energy_total_sensor: boiler_energy_total  # энергия всего (опционально)
    min_temperature: 35.0°C      # минимальная уставка
    max_temperature: 75.0°C      # максимальная уставка
    heat_deadband: 1.0°C         # гистерезис включения
    heat_overrun: 0.5°C          # гистерезис выключения
    default_target_temperature: 50.0°C
    default_power_step: ECO      # ECO / COMFORT / BOOST
    update_interval: 200ms       # интервал обновления
```
